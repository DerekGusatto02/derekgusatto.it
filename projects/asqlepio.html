AsQlepio

  • UniPD
  • Completed

AsQlepio is the project developed for the "Object-Oriented Programming" course at the University of Padua during the academic year 2023/24: the project consists in the design and implementation of a software for managing some sensors.

The project was developed with Veronica Tecchiati between December 2023 and February 2024.

The project consists in the creation of a hierarchy of classes in C++ representing sensors (chosen by the students) and a GUI, to be created with the Qt framework. In this simulation, the end users of the application must be able to define new sensors, modify and delete existing ones, and start a simulation. The use of appropriate design patterns was recommended. The project required approximately 50 hours of total individual work.

Key Aspects

  • Model design: The system uses an abstract base class Sensor with attributes like identifier, name, reliability, and a measurement range. Various sensor types (e.g., Oxygen Saturation, Heart Rate) extend from this base class.
  • Polymorphism: Utilized in the sensor hierarchy and implemented through strategy and visitor patterns, allowing flexible sensor data handling and GUI customization.
  • Data persistence: Data is stored in JSON format, allowing easy saving and loading of sensor data.
  • Implemented features: The project includes five sensor types, sensor creation and modification, simulation visualization, JSON conversion, and search functionality.
  • Development Enviroment: Linux (Ubuntu 22.04 LTS) using GCC, Qt, and qmake.

Project specifications

The project must necessarily satisfy the following constraints:

  1. be an original work by the student or the pair of students
  2. be entirely written in C++
  3. provide a graphical interface created in Qt
  4. compile without errors on the virtual machine provided (warnings during compilation are tolerated, although generally penalized)
  5. implement the principles of encapsulation and information hiding of good object-oriented programming
  6. maintain a clear separation between the logical model and the graphical interface
  7. execute efficiently and robustly, without runtime errors
  8. use polymorphism in a non-trivial way
  9. implement a class hierarchy for sensors with at least five concrete classes for projects carried out in pairs
  10. allow the creation, search, modification and deletion of sensors, these operations must be usable via the graphical interface
  11. implement the persistence of sensor data in a structured format (JSON, XML, etc.) for projects carried out in pairs
  12. allow, via the graphical interface, the saving and loading of a sensor file (as described in the previous point) through appropriate dialog windows: the use of paths is not allowed of wired sensor files in the source code
  13. allow, via a graphical interface, to start a simulation for a selected sensor, simulating a certain number of data collections; the data must then be displayed in an appropriate manner, for example via QtCharts; the simulation does not need to be interactive
  14. be accompanied by a report in pdf format, in Italian or English, of maximum 8 pages with 10pt text.

Abstract

AsQlepio is an advanced medical sensor data management system, focused on the accurate monitoring of vital parameters such as blood pressure, heart rate, respiratory rate and body temperature. Our project aims to provide a comprehensive and intuitive solution for the collection, analysis and visualization of critical health data, aiming to improve the efficiency and accuracy of medical information management. Unlike conventional approaches, AsQlepio stands out for its user-friendly interface and the ability to integrate different types of sensors into a single platform. The main goal is to provide healthcare professionals and patients with a centralized environment for real-time monitoring and historical recording of crucial vital parameters.
